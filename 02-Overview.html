The java file will compile successfully but will not run successfully. It is because, while invoking the java compiler during compilation, the java compiler does not issue a call to the main() method. It simply checks for syntactical errors in code. So this is the reason why such a source file compiles successfully. But, when we execute such a source file then the execution reports an error. It is because, when we execute a java source file, then the jvm issues a call to the main() method, and since in this case, the jvm will not be able to locate the main() method it reports an error. So, the concept to remember is, "main() function is not called during compilation, it is called during execution by the java compiler.

Byte code is a set of instructions that is obtained on compiling a java source code.

".class"

Byte code provides :
1. Portability to java programs
2. Platform independence to java

Byte code is generated on compiling the java source code or by issuing a call to the java compiler.

Java is platform independent/ portable because of the byte code. On compiling a java source file, some ".class" files are generated that contain byte code. Now, since the byte code can be executed on any OS, therefore java is platform independent and java programs are portable.

Byte code is executed by jvm (java virtual machine) when we run our java program.

Byte code gets converted into machine code.

Machine code contains a set of instructions that are written in binary form. This machine code can directly be executed by the OS.

This command issues a call to jvm (java virtual machine) and asks jvm to execute the byte code that is present in the "ABC.class" file. The byte code present in the file "ABC.class" is executed and converted into machine code by the jvm.
