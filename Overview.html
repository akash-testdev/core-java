All languages need to convert the source code into machine code so that this code can be understood and executed by the OS. In Java, this is a 2 step process where as in other programming languages, this is a 1 step process. In Java, the compiler converts the source code into byte code and then in the 2nd step, the jvm converts this byte code into machine code. However, in other languages, the source code is directly converted into machine code by the compiler. So this is where Java is different as compared to other languages w.r.t the process of program execution.

The execution time of a program is nothing but the time in which the source code gets converted into machine code and gets executed by the OS. In the case of most other programming languages like C, the source code on compilation is directly converted into machine code and is executed by the OS thereafter. So, this is the reason why programming languages like C have very low compilation time. On the other hand, in the case of java, the execution time or the time in which the source code is converted into machine code and executed by the OS is more because, the source code is not directly converted into machine code by the compiler. The compiler converts the source code into byte code. This byte code is now converted into machine code by the jvm and then executed by the OS. So, this is the reason why the execution time of java programs is much more as compared to other programming languages.

Advantages :
1. Byte code provides portability to java programs
2. Byte code is responsible for platform independent nature of java

Disadvantage :
1. The presence of byte code in between source code and machine code increase the execution time of java programs thereby compromising on the performance of java

JIT is an acronym for Just-In-Time. JIT is a compiler that is a part of JVM and helps JVM to convert the byte code into machine code in a much lesser time as compared to the case when JIT is not included in JVM. Thus, JIT is a part of JVM that improves the performance of JVM by helping JVM to convert byte code into machine code at a much faster rate.

Although there are many mechanisms that JIT uses for improving the performance of JVM, here is one of the approach - The JIT groups similar instructions (for example, all instrctions involving printing to console or System.out.println() )together and converts them into machine code at the same time (Had there been only JVM, it would have converted even the similar instructions one by one into machine code thereby requiring more time).

Comments are the remarks that are provided by the programmer while writing the program for better understanding of code. The contents of the comments are ignored by the compiler. In Java, we have 3 types of comments -
1. Singleline Comments
2. Multiline comments
3. Documentation comments

1. Singleline Comments - begins with 2 forward slashes and ends with the line
2. Multiline comments - begins with 1 forward slash and 1 star "/*" and ends with 1 star and forward slash "*/" on same or different line
3. Documentation comments - begins with 1 forward slashes and 2 stars "/**" and ends with 1 star and forward slash "*/" on same or different line

The execution of all java programs begin from the main() function of the class that is called by the JVM during execution. It is because, when we run our program using the syntax, java classname, then we are actually issuing a call to JVM. When JVM is called, it issues a call to the main() method present in the class that is run. So this is the reason why the execution of all java programs begin from the main() function.

1 - public : "public" is an access specifier which ensures that the main() method can be accessed by any code outside the class in which it is defined. Now, since the main() method is called by the JVM during the time of executing the class, therefore it is mandatory for us to declare main() method using "public" access specifier.

2 - static : When static is used with a class member, it can be accessed using the name of the class outside the class in which it is defined. Now, we know that the main() method is called by the JVM. The JVM calls the main() method without creating the object of the class that is executed. Therefore, since the main() method is static, the JVM calls it using the class name that is executed. For example, if during runtime, we executed the command as :
> java ABC
then the JVM will issue a call to the main() method as "ABC.main()" (without creating the object of the class ABC)

3 - void : When void is used as a return type for a function, it signifies that the function does not return anything to the caller. Now, since the main() method is called by the JVM, therefore it is pointless to return something to JVM. Also, an attempt to do so is reported as an error by JVM

4 - String args[] : String args[] is an array of strings. We can provide inputs to this array 'args' during the time we execute our program as :
> java ABC Kanpur Lucknow Hyderabad
Here the strings "Kanpur", "Lucknow", "Hyderabad" will be provided as an input to the array args.
These strings are also known as command line arguments.

Command line arguments are the Strings that are passed to the main() function while running the class file. An array of strings named 'args' acts as a parameter to the main() function for accepting these strings.
For ex,
> java ABC Kanpur Lucknow Hyderbad
Here the strings "Kanpur", "Lucknow", "Hyderabad" are called command line arguments.

To understand the modifications that can be done in the declaration of the main() function, we need to understand the following points :
1 - Syntax of method declaration and using multiple modifiers.
2 - Possible modifications in array declaration.
3 - Concept of varargs.

1 - Syntax of method declaration and using multiple modifiers :
Following is the standard syntax for a method declaration :
[modifiers] [return type] [name of the method]([Parameters])
Now, it is not mandatory to use only one modifier while declaring a method. Multiple modifiers can be used for the purpose and they can be used in any order.
In the declaration of main() method, we have 2 modifiers - public and static.
Now since the modifiers can be used in any order, therefore this modification is perfectly valid :
static public void main(String args[])
Note that we cannot drop out 'public' nor we can replace it with 'protected' or 'private'. It is because the main() method is called by JVM which does not lie within the class/package/subclass in which the main() method has been defined.
Note that we cannot drop out static because main() method is called by JVm without creating anuy objects of the class in which the main() method is present. It calls the main() method using the class name. Therefore, static cannot be dropped out as well.

2 - Possible modifications in array declaration :
There are 2 valid ways in which an array can be declared :
int a[];
int[] a;
Therefore, we can modify the parameter for the main() function as :
String[] args
Therefore this is also a valid modification in the declaration of the main() function :
public static void main(String[] args)

3 - Concept of varargs :
Varargs is an acronym for Variable Length arguments.
A variable length argument is specified using three dots "...".
Therefore this is also a valid modification in the declaration of the main() function :
public static void main(String... args)

Also, note that any permutations and combinations of the above allowed modifications are also valid. For example,
static public void main(String... args)
is perfectly valid.
